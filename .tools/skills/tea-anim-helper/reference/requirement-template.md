# 用户需求模板规范

本文档定义**让用户描述一段动画**所需提供的信息，以及 AI 如何引导用户补全。目标：把模糊的自然语言收敛成可直接落地为 `_Tick` 动画的结构化需求。

## 1. 最小模板（单 bmp 单行）

```
bmp-<bmpId>&npc-<srcId>(<x>,<y>,<w>,<h>): <面向时间的生成与变化的自然语言描述>
```

示例：

```
bmp-2&npc-1(0,0,128,128): 缩放为刚好覆盖整个屏幕, 在 0~1s 内透明度从 0 变为 1.
```

解读：

- `bmp-2`：申请一个 **bmp id = 2** 的实例（注意：实际脚本里这个 id 会被换算成相对 `BmpIdOffset` 的偏移，见下文「关于 bmpId」）。
- `npc-1`：贴图资产取 **id 为 1 的 npc**。
- `(0,0,128,128)`：从该资产以左上角 `(0,0)` 为起点，裁剪 `128x128` 的区域作为该 bmp 的图像（资产多为图集，按区域截取很常见）。
- `: ...` 之后：描述这个 bmp 在运行时**怎样随时间生成与变化**。

## 2. 用户必须说清的三件事

| # | 维度 | 含义 | 例 |
| --- | --- | --- | --- |
| 1 | **bmp ↔ npc 选用与截取** | 每个 bmp 用哪个 npc 资产、截取 `(x,y,w,h)` | `npc-1(0,0,128,128)` |
| 2 | **面向时间的生成** | 这个 bmp 何时出现 / 申请（开场就有？某帧才申请？） | `0s 出现` / `2s 时才申请` |
| 3 | **面向时间的变化** | 位置 / 缩放 / 旋转 / 透明度 / 颜色 / 帧 随时间如何变 | `0~1s 透明度 0->1`、`沿曲线飞入`、`持续旋转` |

## 3. 帧动画（图集序列帧）

若某个截取区域是**序列帧图集**，用户还需说明：

```
bmp-3&npc-1(128,128,128,128) [anim: frames=4, cols=2, loopFrom=0, fps=12]
```

- `frames`：总帧数。
- `cols`：图集横向每行的帧数（`BmpStoreAnimInfo(cnt, cellCntX)` 的第二参）。
- `loopFrom`（可选）：尾循环帧数，`<=0` 表示整体循环；见 `BmpStoreAnimLoop`。
- `fps`（可选）：播放速度，换算成每帧推进量 `step = fps / 60`。
- 若帧之间有像素间隔，用 `interval=(ix,iy)` 补充（`BmpStoreAnimInterval`）。

## 4. 强烈建议补充（有则动画质量更高）

- **创作意图**：这段动画想表达什么（如「飞碟坠毁前的高速旋转飞入」）。
- **详细呈现效果**：分阶段/分镜描述，含节奏与重点帧。
- **对标动画**：参考了哪段已有演出 / 视频 / 游戏 CG。
- **风格**：卡通 / 写实 / 故障感 / 复古像素…
- **总时长**：明确秒数或帧数（缺省时 AI 取合理值并标注，如 60 帧）。

## 5. 多 bmp / 多阶段模板

复杂动画可列多行 + 一段时间轴说明：

```
# 动画名: DiscFly
# 时长: 10s (600 帧)
# 风格: 复古像素, 故障感

bmp-0&npc-2(3984,0,8,8) [anim: frames=4, cols=2]   ' 飞碟本体, 会随尺寸切换不同精度图
bmp-1&npc-1(768,256,1,1)                            ' 纯色背景片(拉伸用)

时间轴:
  0~~? : 飞碟从屏幕上方沿贝塞尔曲线飞入, 同时由小变大、由快转慢旋转
  约 5s: 背景片骤然放大铺满全屏(转场白闪), 记录切换时刻
```

> 这种「先列各 bmp，再给时间轴分镜」的写法便于把复杂演出落成单脚本，可参考本 skill 的 `examples/starfield.tea`。

## 6. AI 引导话术（信息不足时）

当用户描述过于笼统（例如「做个炫酷的爆炸」），AI 应回贴本模板并按缺口逐项追问，例如：

> 为了把它做成可复用的 tick 动画，请补充：
> 1. 用哪个 npc 资产做爆炸贴图？截取区域 (x,y,w,h)？是序列帧吗（帧数 / 排布 / 速度）？
> 2. 爆炸出现在屏幕什么位置、什么时刻？
> 3. 期望时长（秒）？要不要伴随震屏 / 闪白 / 碎片飞溅？
> 4. 有对标的爆炸效果或风格吗？

若用户希望快速出原型，AI 可**先按合理默认值生成**，在脚本头部「调参区」用注释标注假设，便于其后微调。

## 7. 关于 bmpId（重要）

用户模板里写的 `bmp-2` 是**直观编号**。真正落到脚本里时，为避免多动画并存撞车，**所有 bmp id 都相对 `_Launch(BmpIdOffset)` 的偏移分配**：

- 模板 `bmp-0` → 脚本 `BmpIdOffset + 0`
- 模板 `bmp-1` → 脚本 `BmpIdOffset + 1`
- ……

即把模板里的编号视作「相对偏移」。AI 在生成脚本时务必如此换算，并在头部维护一张「bmp 分配表」注释。详见 [`animation-contract.md`](animation-contract.md)。
